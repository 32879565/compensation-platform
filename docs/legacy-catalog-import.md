| 项目 | 真实数据范围 |
|---|---:|

原始缓存和 Excel 含员工敏感信息，只用于本地迁移与对账，禁止提交到 Git、制品或日志。

## 旧数据能证明什么

旧工资明细可以证明某个字段、职位和金额在特定月份真实出现过，例如：

- 薪资基数：`综合薪资`、`基本薪资`；
- 岗位金额：`服务岗`、`传菜岗`、`前厅领班岗` 等；
- 补贴：`房补`、`通岗补贴`、`留守休息补贴`、`化妆/饮品补贴` 等；
- 奖励：`留年激励`、`宿舍长奖励`、`外卖奖励` 等；
- 扣减：`水电费`、`空调费`、`考勤扣罚`、`绩效扣除` 等。

旧系统没有正式的组件编码、固定/浮动属性、计税/社保/公积金口径，也没有官方职级和薪档主数据。因此系统不会把历史字段或历史薪资分位自动当作现行政策。

## 导入流程

1. 使用隔离的新目标库并完成备份，先将数据库升级到 S6 薪资记录结构。不能先升到 `head`，因为后续 D2/D3 是依赖已加载历史记录的数据迁移：

   ```powershell
   cd backend
   .venv\Scripts\alembic upgrade 089dead90284
   ```

   此时数据库停留在 S6，不能启动会自动执行 `upgrade head` 的当前应用或依赖当前组织管理页面。应从受控备份恢复组织主数据；若这是全新隔离库且尚无集团根组织，可由数据库管理员执行下面的 S6 兼容初始化（正式编码和名称须先由业务负责人确认）。D2 会把无法匹配的历史门店放到该集团下的“历史门店（待归属）”区域：

   ```sql
   INSERT INTO org_unit
       (parent_id, type, name, code, city, status,
        created_at, updated_at, created_by, is_deleted, deleted_at)
   SELECT
       NULL, 'GROUP', '集团总部', 'GROUP-ROOT', NULL, 'ACTIVE',
       now(), now(), NULL, false, NULL
   WHERE NOT EXISTS (
       SELECT 1 FROM org_unit WHERE type = 'GROUP' AND is_deleted = false
   );
   ```

   导入前必须确认下列查询返回 `1`；返回 `0` 或大于 `1` 都要先由数据库管理员修正组织主数据，不能继续迁移：

   ```sql
   SELECT count(*)
   FROM org_unit
   WHERE type = 'GROUP' AND is_deleted = false;
   ```

2. 将旧缓存迁入 `salary_record`；迁移程序只读旧 SQLite，并以 `HISTORICAL` 标记来源：

   ```powershell
   cd backend
   .venv\Scripts\python -m app.importing.migrate_legacy --sqlite "D:\backup\salary_data.sqlite"
   ```

3. 数据写入成功后再升级到最新版本。D2 会创建并认领缺失的历史门店，D3 会创建带 `LEGACY-NAME-` 标识的临时员工并回填可确认的历史身份：

   ```powershell
   .venv\Scripts\alembic upgrade head
   ```

4. 核对 `salary_record` 的记录数、月份、`org_unit_id`，并复核临时员工。未取得稳定 `employee_id` 或工号的记录不会进入职级薪资分位样本。
5. 使用集团级人事账号进入“薪资组件”或“职级体系”，点击“审阅旧系统真实数据”。
6. 系统只展示聚合证据。人事为组件补齐编码、类型、固定/浮动及计税口径后确认创建。
7. 历史职位仅作为核对证据。人事必须另行填写正式职级编码、级别和现行薪档，并进行政策确认。

## 强制安全边界

- 预览只读取 `HISTORICAL` / `IMPORT` 数据，不读取新系统核算结果。
- 预览不返回姓名、工号或单条工资；少于 5 名独立员工的职位聚合整项隐藏，同一员工跨月记录不能叠加达到阈值，零值薪资占位不计入样本。
- `import:run` 与对应的组件/职级写权限都必须来自集团级授权，局部门店权限不能组合放大。
- 预览、导入确认和目录应用共用事务锁；确认时校验来源快照，任何来源变化后必须重新预览。
- 同一旧字段或历史职位只能认领一次，不能通过更换编码重复创建。
- 每次预览和应用均写入追加式审计，保留来源快照、候选聚合、覆盖月份、完整组件/职级口径、确认人和确认理由。
- `出勤工资`、`法定补贴`、`社保`、`公积金`、`个税`、`押金`、`应发工资`、`实发工资` 等派生或法定项目不得作为员工薪资结构组件导入。

## 验收对账

迁移完成后至少核对：总记录数、月份范围、各月行数、门店匹配数及合计工资总额。任何无法解析的月份、门店或金额都会使迁移整体失败，不做静默丢弃或归零。
